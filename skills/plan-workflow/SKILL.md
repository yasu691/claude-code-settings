# 実装計画 Skill

## 目的

実装前に、要件・影響範囲・リスクを整理して手戻りを減らす。

## 実行手順

1. 要件を再定義し、成功条件を明文化する
2. 変更対象のファイル・機能を洗い出す
3. 実装を Phase 単位で分割する
4. テスト戦略（単体/結合）を決める
5. リスクと緩和策を列挙する

## 出力フォーマット

```markdown
# 実装計画: <テーマ>

## 要件
- ...

## 影響範囲
- <path>: 変更理由

## 実装ステップ
1. ...
2. ...

## テスト戦略
- 単体:
- 結合:

## リスクと対策
- リスク:
  - 対策:
```

## ルール

- この Skill ではコードを書かない
- 不明点は推測せず前提として明記する
