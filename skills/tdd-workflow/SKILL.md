# TDD Skill

## 目的

実装品質を担保しながら、過剰実装と回帰を防ぐ。

## 手順

1. テストケースを先に定義する
2. 失敗するテストを書く（RED）
3. 最小実装で通す（GREEN）
4. ふるまいを維持して整理する（REFACTOR）
5. カバレッジを確認する（目標80%以上）

## チェック観点

- 正常系と異常系の両方があるか
- 外部依存は適切に隔離されているか
- テスト名から期待動作が読み取れるか

## 禁止事項

- テストを書かずに先に実装すること
