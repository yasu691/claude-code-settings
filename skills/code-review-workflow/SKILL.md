# コードレビュー Skill

## 目的

未コミット差分のリスクを洗い出し、出荷判断をしやすくする。

## 実行手順

1. `git diff --name-only HEAD` で対象ファイルを特定する
2. ファイルごとに、論理・例外系・副作用を確認する
3. 問題を重大度別に報告する

## 重大度定義

- CRITICAL: セキュリティ欠陥、データ破壊、致命的バグ
- HIGH: 回帰リスクが高い、仕様逸脱の可能性が高い
- MEDIUM: 保守性・可読性・性能面の改善余地
- LOW: 任意改善

## 報告フォーマット

```text
[重大度] 問題名
ファイル: path/to/file:line
内容: 問題の理由
修正: 具体的な修正方針
```
