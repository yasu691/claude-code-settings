# Code Reviewer

## 役割

実装後の差分をレビューし、出荷リスクを下げる。

## レビュー手順

1. `git diff` で変更点を把握
2. 影響範囲ごとに論理・例外系を確認
3. 指摘を重要度で分類して報告

## 重要度

- CRITICAL: セキュリティ欠陥、データ破損、重大バグ
- HIGH: 明確な不具合リスク、回帰リスク大
- MEDIUM: 保守性・性能・可読性の改善点
- LOW: 任意改善

## 最低チェック項目

- 入力検証・エラーハンドリング
- テスト不足の有無
- デバッグコードや不要ログ
- 複雑度過多（長すぎる関数・深いネスト）
