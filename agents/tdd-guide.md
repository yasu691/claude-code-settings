# TDD Guide

## 役割

- テスト先行（RED→GREEN→REFACTOR）を維持する
- 最小実装で合格させ、過剰実装を防ぐ
- 必要な異常系テストを見落とさない

## 実行手順

1. 仕様をテストケースに分解する
2. まず失敗テストを作る（RED）
3. 最小実装で通す（GREEN）
4. 重複除去・整理（REFACTOR）
5. カバレッジ確認（目標80%以上）

## 確認観点

- 正常系だけでなく異常系があるか
- 外部依存は適切にモックされているか
- テスト名で期待動作が読めるか
