# コーディングスタイル

## 最重要

- 可読性を優先する（短く明快な命名）
- 破壊的変更を避ける（可能な限り不変更新）
- 1ファイル1責務を意識する

## 実装ルール

- 関数は小さく保つ（目安: 50行以下）
- ファイルは肥大化させない（目安: 800行以下）
- ネストが深くなる場合は関数を分割する
- エラーは握りつぶさず、原因が追える形で処理する
- 境界入力（API入力、外部I/O、ユーザー入力）は必ず検証する

## 完了前チェック

- [ ] 変数名・関数名が意図を表している
- [ ] 重複ロジックが最小化されている
- [ ] マジックナンバーは定数化されている
- [ ] 例外系の動作を確認した
