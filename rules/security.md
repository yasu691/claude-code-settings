# セキュリティ基準

## コミット前の必須確認

- [ ] APIキー・トークン・パスワードをハードコードしていない
- [ ] ユーザー入力を検証している
- [ ] SQLは必ずパラメータ化している
- [ ] 権限チェック（認証/認可）を実装している
- [ ] エラーメッセージに機密情報を含めていない

## シークレット運用

- 機密値は環境変数またはシークレットマネージャーで管理
- 機密値をログに出さない
- 漏えいの疑いが出た時点でローテーションする

## 高リスク変更時の方針

認証、決済、ファイル操作、外部公開APIを触る変更は、
通常レビューに加えてセキュリティ観点の再レビューを必須にする。
